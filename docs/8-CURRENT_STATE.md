## 当前阶段

BestBlogs 当前主线是 **AI 驱动的私人阅读助手**。近期进入 BestBlogs 3.0 设计启动期，优先验证 PMF 和用户真实需求，核心方向是「主题早报」。

1. 稳定公共质量池，让每日早报、精选周刊、主题解读和内容广场持续产出高质量内容。
2. 设计主题早报，让用户可以通过自然语言定制多份不同目的的早报。
3. 验证 Pro 阅读工作流是否真的服务用户的长期高质量信息输入。

当前仍以「我的早报」相关使用作为重要观察入口，但 3.0 会进一步关注多份主题早报的创建、持续打开、调整和留存表现。

## 产品结构

### 公共策展层

公共策展层面向所有访问者，承担 BestBlogs 的质量判断展示：

- **每日早报**：每天输出重点内容摘要和阅读线索。
- **精选周刊**：按周回顾高质量内容。
- **主题解读**：把碎片内容整理成事件、领域、人物或产品视角。
- **内容广场**：浏览公开内容池，支持筛选和发现。

这一层坚持「质量优先于数量」，不以热度、点击率或停留时长作为唯一排序依据。

### 我的空间

我的空间面向登录用户，Pro 解锁完整私人阅读助手工作流：

- **我的早报 / 主题早报**：结合关注源、兴趣、阅读行为和自然语言主题设定，生成一份或多份个性化早报。
- **我的关注**：统一管理 RSS、Newsletter、社交平台、视频和播客等来源。
- **我的阅读**：沉淀书签、划线、历史和 AI 辅助理解结果。
- **我的回顾**：把一段时间内的阅读足迹整理成可回看的总结。

这一层的目标不是让用户读更多，而是让用户更快判断什么值得读，并保留自己的思考痕迹。

## 近期重点

1. **主题早报设计**：定义自然语言配置、多份早报、解释、调整和发送节奏。
2. **PMF 与需求验证**：优先判断用户是否真的需要持续跟踪多个主题，而不是先扩张端侧能力。
3. **个性化早报质量**：继续优化候选池、排序、解释和冷启动体验。
4. **关注源管理**：降低添加来源、整理来源和理解来源质量的成本。
5. **Pro 转化路径**：让付费边界清晰可感，不通过打扰式触达推动转化。
6. **读见国内部署版本**：保留 `dujian-app` 作为后续国内部署和本地化验证载体。

## 阶段路线图

### Phase 1：个性化阅读工作流与 PMF 验证

当前主阶段。重点是让用户从「被动浏览内容」转向「主动维护自己的高质量阅读流」，并验证主题早报是否能成为更强的 Pro 价值承载。

阶段成立的信号：

- Pro 用户持续打开「我的早报」或「主题早报」；
- 用户愿意创建、保留并调整多份主题早报；
- 关注、阅读、收藏、划线、不感兴趣等行为能稳定改善推荐；
- Pro 续费和留存由真实阅读工作流驱动，而不是短期新鲜感；
- 反指标健康，没有通过骚扰式触达、标题党或无限滚动拉高指标。

### Phase 2：阅读沉淀深化

在 Phase 1 稳定后，重点转向让阅读结果可回看、可复用、可迁移：

- 更好的个人图书馆；
- 更结构化的划线、笔记和回顾；
- 跨内容的主题研究和对比；
- 更清楚的兴趣变化解释。

### Phase 3：可调用知识基础设施

长期方向是让 BestBlogs 成为外部工具和智能体可调用的高质量阅读基础设施：

- 稳定 OpenAPI；
- 结构化内容包；
- Agent-friendly 查询和阅读接口；
- 与用户授权的外部工具连接。

## 质量红线

所有阶段都受 `1-VISION.md` 的产品红线约束：

- 不以时间消耗为目标；
- 不替用户完成阅读、理解、思考和判断；
- 公共策展层质量门槛不向大众化让步；
- 不以量取胜；
- 不做泛化 AI 工具箱。

当指标增长与这些红线冲突时，以红线为准。
